# Shadows in Radix Themes

Shadows in Radix Themes are derived from a **6-step scale**, providing a consistent and cohesive style across components.

---

## Shadow Scale

- **Step 1**: Smallest shadow (e.g., subtle inset shadows).
- **Step 6**: Largest shadow (e.g., for prominent overlays).

The specific values for each step can be found in the source code, offering flexibility to match your design needs.

---

## Shadow Tokens

CSS variables are available to access shadow values, ensuring uniformity across your theme.

```css
/* Inset shadow */
var(--shadow-1);

/* Shadows for classic panels, like Card */
var(--shadow-2);
var(--shadow-3);

/* Shadows for smaller overlay panels, like Hover Card and Popover */
var(--shadow-4);
var(--shadow-5);

/* Shadows for larger overlay panels, like Dialog */
var(--shadow-6);
```

---

### Usage

These tokens can be applied to custom components, maintaining consistency with the predefined shadows used in the theme.

#### Example:

For a `Card` component:

```css
.card {
  box-shadow: var(--shadow-3);
}
```

For an overlay `Dialog`:

```css
.dialog {
  box-shadow: var(--shadow-6);
}
```

### Notes:

- **var(--shadow-1)** is ideal for subtle inset effects.
- **var(--shadow-4)** and **var(--shadow-5)** are designed for smaller overlay components.
- **var(--shadow-6)** provides depth for larger overlays like modals or dialogs.

By leveraging these tokens, you can achieve a consistent and polished look throughout your application.
