# Container Component in Radix Themes

The `Container` component is used to constrain the maximum width of page content. It is based on the `<div>` element and supports common margin props.

---

## Example Usage

```jsx
<Box
  style={{
    background: "var(--gray-a2)",
    borderRadius: "var(--radius-3)",
  }}
>
  <Container size="1">
    <DecorativeBox>
      <Box py="9" />
    </DecorativeBox>
  </Container>
</Box>
```

---

## API Reference

### Unique Props

- **`asChild`**: If `true`, the `Container` renders a child component instead of the default wrapper.
- **`size`**: Defines the maximum width of the container. Accepts responsive values `"1"`, `"2"`, `"3"`, or `"4"`. Defaults to `"4"`.
  - `"1"`: 448px (`var(--container-1)`)
  - `"2"`: 688px (`var(--container-2)`)
  - `"3"`: 880px (`var(--container-3)`)
  - `"4"`: 1136px (`var(--container-4)`)
- **`display`**: Controls the CSS `display` property. Options are `"none"` or `"initial"`.
- **`align`**: Sets the alignment of content within the container. Options are `"left"`, `"center"`, or `"right"`.

---

## Shared Props

The `Container` component shares several properties with `Box`, `Flex`, `Grid`, and `Section` components. These include:

### Spacing Props

- **Padding**: Use `p`, `px`, `py`, `pt`, `pr`, `pb`, and `pl` to control padding.

### Size Props

- **Dimensions**: Use `width`, `minWidth`, `maxWidth`, `height`, `minHeight`, and `maxHeight` to specify the size.

### Positioning Props

- **Position**: Use the `position` prop for absolute, relative, or fixed positioning.
- **Offsets**: Utilize `inset`, `top`, `right`, `bottom`, and `left` for precise placement.

### Overflow Props

- **Overflow Control**: Manage content overflow using `overflow`, `overflowX`, and `overflowY`.

---

## Notes

- **Responsive Design**: Most props accept responsive values, allowing content to adapt across breakpoints.
- **Customization**: Default sizes can be customized by overriding the respective CSS variables.
